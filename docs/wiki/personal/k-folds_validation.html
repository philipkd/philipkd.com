k-folds validation

Process:

  1. split samples randomly into k-buckets
  2. select one fold as a hold out (validation set), use the rest as a training set
  3. run for n epochs and save your model
  4. re-run with the other folds from scratch and save those models (key point: if you don't start from scratch, then you'll completely fit the training set)
  5. compare those models

stratified k-folds is the same thing, but in such a way that it preserves a representative sample from a certain class

out-of-fold predictions are predictions collected from each of the folds' validation phases. You should have one prediction for each sample, but from when they were in temporarily held out
