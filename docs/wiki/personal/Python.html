argparse

args = parse_arguments(['--flag','1'])

parse_arguments() pulls from sys.argv

parse_arguments(['--flag','1']) pulls from just the array and is equivalent to python filename.py --flag 1

But both combine with the preset arguments.

array notation

a[start:stop] # items start through stop-1
a[start:] # items start through the rest of the array
a[:stop] # items from the beginning through stop-1
a[:] # a copy of the whole array

a[-1] # last item in the array
a[-2:] # last two items in the array
a[:-2] # everything except the last two items

2D array selection

a[:][2] is the same as a[2] because python evaluates the braces in sequence, i.e., a[:] is equivalent to a. For 2D array selection, use numpy.

arrays vs. lists

Arrays are lists with extra features, such as the ability to run mathematical operations over the whole list. Because the extra features depend on the implementation, the key difference is that arrays have to be imported, usually by Numpy

GPU Stats

Snippet that has comparable GPU stats to Kaggle:

import subprocess as sp _output_to_list = lambda x: x.decode('ascii').split('\n')[:-1] ACCEPTABLE_AVAILABLE_MEMORY = 1024 COMMAND = "nvidia-smi --query-gpu=memory.used --format=csv" memory_used_info = _output_to_list(sp.check_output(COMMAND.split()))[1:] memory_used_values = [int(x.split()[0]) for i, x in enumerate(memory_used_info)] print("GPU MEM: %0.1f / 15.9 GB" % (memory_used_values[0] / 1000))
